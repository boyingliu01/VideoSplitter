本仓库采用「底层抛出结构化异常 → 中间层捕获并记录日志 → 顶层通过 Signal/结果字典向 UI 反馈」的分层错误处理策略，覆盖 ffmpeg-skill、video_splitter 核心与 gui 三层。

## 1. 系统与方法
- 自定义异常：ffmpeg-skill/__init__.py 定义 FFmpegError(Exception)，作为所有 FFmpeg 子进程调用失败的统一异常类型；参数校验使用 ValueError，文件缺失使用 FileNotFoundError。
- 标准异常 + 日志：video_splitter/pipeline.py 在 Pipeline.run() 中用 try/except Exception as e 捕获全链路错误，写入 logging.error 并将错误信息放入返回 dict 的 status: "error" / error 字段，再重新 raise 给 CLI。
- Qt Signal 传递：GUI 层不直接抛异常到主线程，而是通过 PySide6.QtCore.Signal 将错误字符串广播给 UI：ReviewController.error = Signal(str)、TranscribeWorker.error = Signal(str)。
- CLI 容错：cli.py 的 cmd_batch 对每个视频单独 try/except，失败时记录 logger.error 并继续处理下一个，最终汇总 OK/Failed 计数。

## 2. 关键文件与位置
- ffmpeg-skill/__init__.py：定义 FFmpegError，在 _run_command、_check_ffmpeg、get_video_info 等处将 subprocess.TimeoutExpired、FileNotFoundError、JSON 解析错误等统一包装为 FFmpegError。
- video_splitter/pipeline.py：Pipeline.run() 用 try/except 包裹 precheck→extract→transcribe→chapter→validate→cut 全流程，失败时设置 result["status"]="error" 并 raise。
- video_splitter/cli.py：cmd_batch 逐个视频 try/except 并 logger.error；cmd_gui 捕获 ImportError 给出安装提示后 sys.exit。
- gui/controllers/review_controller.py：通过 self.error.emit(str(e)) 把保存/导出时的异常传给 UI；索引越界等逻辑错误以 Signal 形式通知而非抛异常。
- gui/workers/transcribe_worker.py：run() 内 try/except 捕获 ASR 错误，self.error.emit(str(exc)) 回传主线程。

## 3. 架构与约定
- 异常上抛原则：ffmpeg-skill 和 video_splitter 内部只 raise Python 内置异常或 FFmpegError，不在业务层吞掉异常；上层（pipeline、CLI、GUI）负责捕获、记录、转化。
- Signal 替代 return-code：GUI 异步任务一律通过 error Signal 报告失败，避免跨线程直接 raise 导致崩溃。
- 批量容错：批处理命令对单个失败不中断整体流程，收集结果后统一输出摘要。
- 无 panic/recover：Python 侧未使用 try/finally 之外的 recover 模式；资源清理（如临时 WAV 删除）放在 try/except OSError 中静默忽略。

## 4. 开发者应遵守的规则
1. 底层函数：对外暴露的 API 遇到外部工具失败时 raise FFmpegError；参数非法 raise ValueError；文件不存在 raise FileNotFoundError。不要 catch 后返回 None。
2. 中间层编排：在 pipeline 级别用 try/except 捕获异常，记录 logger.error，并在返回结构中标注 status="error"，然后重新 raise 让调用方决定如何处理。
3. GUI 异步任务：worker 的 run() 方法必须包裹 try/except，失败时 self.error.emit(str(exc))；成功时 self.finished.emit(result)。禁止在主线程 raise。
4. CLI 批处理：单任务失败仅记录 logger.error 并继续，最后汇总统计。
5. 不要吞异常：除非是明确的「可恢复」场景（如删除临时文件），否则不要 bare except: 或 except Exception: pass。
