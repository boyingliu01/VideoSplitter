模块以 `__init__.py` 中的 `FFmpegSkill` 单类为核心，所有业务逻辑集中于此：
- 依赖注入：构造函数接受 `ffmpeg_path` / `ffprobe_path`（默认走 PATH），通过 `_check_ffmpeg` 在实例化时探测可执行文件。
- 命令执行：统一的 `_run_command` 使用 `subprocess.Popen` 启动 ffmpeg，逐行读取 stderr 并通过正则 `time=HH:MM:SS.xx` 触发可选的 `progress_callback`；错误统一包装为自定义 `FFmpegError`。
- 元数据查询：`get_video_info` / `_get_duration` 调用 ffprobe 并以 JSON 解析返回结构化信息。
- 高层 API：`convert_format`、`resize`/`resize_to_preset`、`cut`、`extract_audio`、`add_watermark`（支持图片叠加与 drawtext 文本水印）、`merge_videos`（基于 concat filter_complex）、`adjust_quality` 等方法各自拼装 ffmpeg 参数列表后委托给 `_run_command`。
- CLI 外壳：`ffmpeg_tool.py` 通过 argparse 子命令（convert/resize/cut/extract-audio/watermark/merge/quality/info）将命令行参数映射到 `FFmpegSkill` 方法，作为独立可运行脚本或被 skill 框架调用。
- 辅助文件：`examples.py` 演示各 API 用法；`tests.py` 基于 pytest，要求系统 PATH 中存在 ffmpeg/ffprobe，并用临时目录构造假输入验证参数校验路径。
依赖方向：CLI → `FFmpegSkill` → 外部 `ffmpeg`/`ffprobe` 进程，模块内部无其他子包。
