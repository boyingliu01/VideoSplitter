根目录按「技能 → 核心 → 界面」三层装配：
- ffmpeg-skill 暴露纯 Python 封装的 FFmpeg 调用（convert_webm、resize_video、get_info），被 gui 与 video_splitter 共同复用；
- video_splitter 作为无头核心，通过 Pipeline 编排 audio→transcribe(chapter/validator)→cutter(subtitle_burner)，对外仅暴露 CLI 与 review 接口；
- gui 是唯一的用户入口，其 controllers/workers/widgets 组合 ffmpeg-skill 与 video_splitter.review，把异步任务挂到 QThread 并通过信号槽回传进度。
三者之间不存在反向依赖：gui 不直接操作 ffmpeg，video_splitter 也不感知 GUI，所有跨模块通信都通过函数调用与数据类（如 ReviewResult）完成。
