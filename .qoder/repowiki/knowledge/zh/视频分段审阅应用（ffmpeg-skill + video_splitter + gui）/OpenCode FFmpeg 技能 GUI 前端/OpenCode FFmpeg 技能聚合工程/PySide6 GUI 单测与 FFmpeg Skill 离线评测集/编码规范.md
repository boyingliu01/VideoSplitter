- 测试文件通过 `os.path.dirname(os.path.dirname(...))` 计算项目根目录并 `sys.path.insert(0, ...)`，从而直接 import 非包路径下的 `gui.*` 源码。
- 对外部子系统（如 `create_engine`）一律使用 `unittest.mock.MagicMock` 配合 `@patch` 或运行时替换模块属性进行隔离，避免真实模型加载。
- FFmpeg skill 评测采用 `with_skill` / `without_skill` 双分支目录结构，每个分支内再分 `run-1` 等编号子目录，便于多轮回归对比。
- 每个评测断言以 `{text, passed, evidence}` 字典形式记录，统一通过 `check_assertion` 构造，最终聚合到 `expectations` 列表输出为 JSON。
