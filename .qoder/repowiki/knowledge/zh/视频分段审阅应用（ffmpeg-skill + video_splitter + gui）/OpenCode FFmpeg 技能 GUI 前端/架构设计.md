GUI 层按 MVC 风格分层组织：`app.py` 中的 `MainWindow` 作为 Qt 应用入口，负责构建菜单栏、中央 QSplitter（左侧 VideoPlayerWidget + 右侧 QTabWidget）和状态栏，并集中连接各组件信号；`controllers/` 下的 ReviewController/SplitController 是纯业务层 QObject，维护当前片段索引与已修改集合，封装 transcript/progress 的加载、保存与 SRT 导出，通过 `segment_changed`/`error` 等 Signal 与 UI 解耦；`widgets/` 下三个自定义 QWidget（VideoPlayerWidget、SubtitlePanel、StatusBarWidget）仅负责展示与用户输入，不持有业务状态；`workers/` 将耗时 ASR 调用封装为可在 QThread 中运行的 QObject，通过 progress/finished/error 信号回传结果，避免阻塞主线程。依赖方向单向：MainWindow → Controller + Worker + Widgets；Controller 仅依赖 `video_splitter.review` 与 `video_splitter.extractor.transcribe`，不反向引用 GUI。
