本模块由三个相互独立的子目录组成：
- `tests/`：基于 pytest 的单元测试套件，`conftest.py` 将仓库根目录注入 `sys.path`，使测试可直接 import `gui.workers.*`；`test_workers.py` 使用 `unittest.mock.MagicMock` + `@patch` 模拟 `create_engine`，验证 `TranscribeWorker` 的信号发射与 QThread 集成。
- `ffmpeg-video-workspace/`：FFmpeg skill 的离线评测工作区。顶层 `evals.json` 声明 3 个评测用例（convert-format / resize-video / video-info），`grade_all.py` 通过调用系统 `ffprobe` 对每个 `iteration-1/<eval>/<with_skill|without_skill>/outputs` 下的产物执行断言，并将结构化结果写入 `grading.json`；`benchmark.json` 汇总 with/without skill 两组的 pass_rate、耗时、token 等统计及 delta；`fix_grading.py`、`convert_webm.py` 为辅助修复/转换脚本；`test-files/test_input.mp4` 提供固定输入视频。
- `skill-cert-results/`：存放外部 skill 认证引擎产出的 `SKILL-result.json` 与缓存，包含 L1-L8 指标、token 分析、drift 与 judge 原始 trace，仅作为只读结果存档。
依赖方向单向：评测脚本 → 被测代码（通过 sys.path 注入）→ 系统 ffprobe；认证结果不被任何脚本消费。
