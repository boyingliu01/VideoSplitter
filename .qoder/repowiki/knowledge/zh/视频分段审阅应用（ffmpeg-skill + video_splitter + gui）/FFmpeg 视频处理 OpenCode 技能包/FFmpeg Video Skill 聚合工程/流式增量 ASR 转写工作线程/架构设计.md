本模块为 GUI 层的 Qt 工作线程对象（继承 `PySide6.QtCore.QObject`），通过 `@Slot` 暴露的 `run(video_path)` 作为 QThread 入口，内部以状态机四阶段执行：
- Phase 1：并行获取视频时长（ffprobe）、启动 FFmpeg 子进程全量抽取 WAV、加载 FunASR 模型；完成后发出 `audio_ready`。
- Phase 2：基于 `FUNASR_CHUNK_SECONDS` 计算分块数，构建有序 `deque` 队列。
- Phase 3：循环从队列取块，调用 `_extract_audio_range` 切出子段，经 `model.generate` + `FunASREngine._extract_segments` 识别后偏移时间戳去重，逐块 emit `segments_ready` / `chunk_completed`；每 3 块触发一次 `gc.collect()` 控制内存。
- Phase 4：清理临时文件并发出 `transcription_complete`。

跨线程同步仅依赖 CPython GIL 保证原子性的两个字段：`_priority_chunk_index` 与 `_cancelled`，由 GUI 线程通过 `request_priority(time_seconds)` 和 `cancel()` 修改，worker 在每块边界检查。依赖方向单向指向 `video_splitter.extractor.engines`（FunASR 引擎/FFmpeg 工具）与 `video_splitter.config.SplitConfig`，不反向引用 GUI。
