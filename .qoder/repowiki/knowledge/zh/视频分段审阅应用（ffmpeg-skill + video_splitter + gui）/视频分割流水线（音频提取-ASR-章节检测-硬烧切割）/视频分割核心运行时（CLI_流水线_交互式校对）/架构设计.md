本模块由三个职责清晰的文件组成，构成用户可见的运行层：
- `cli.py` 基于 `argparse` 子命令注册 `split/transcribe/cut/check/review/gui/batch` 六个入口，每个子命令对应一个 `cmd_xxx` 函数；通过 `SplitConfig.from_env()` 读取配置并构造 `Pipeline`，属于纯调度层，不持有业务状态。
- `pipeline.py` 定义 `Pipeline` 类，按固定顺序串联 `AudioExtractor → transcribe → ChapterDetector → ChapterValidator → VideoCutter`，并通过 `.transcript.json` / `.chapters.json` 中间文件实现 `resume` 断点续跑；`dry_run` 仅估算 token 与费用，不调用 LLM。
- `review.py` 提供无外部依赖的交互式校对循环，使用 `<video_path>.review_progress.json` 持久化当前段落索引与修改计数，支持 `:q/:p/:j N/:h` 等命令，并在结束时原子写入 transcript 并调用 `to_srt` 重建 SRT。
依赖方向单向向外：`cli → pipeline + review`，`pipeline` 再向下依赖 `extractor/analyzer/splitter/config`；`review` 仅在完成时按需 `from .extractor.transcribe import to_srt` 避免循环导入。
